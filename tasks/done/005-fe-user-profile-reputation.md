# Профили пользователей, карма и репутация (Frontend)

## Цель (Value)
- **Для пользователя:** Дать ощущение прогресса и признания в сообществе. Мотивировать на создание качественного контента через систему достижений и видимой репутации.

## Требования (Requirements)
1.  **UI профиля:** Реализовать UI-страницу профиля, где наглядно отображается следующая информация:
    -   Текущая карма пользователя.
    -   Список полученных достижений.
    -   Лента активности пользователя.
2.  **Интеграция с API:** Интегрировать страницу профиля с backend API для получения и отображения актуальных данных.

## Критерии приёмки (Acceptance Criteria)
- Страница профиля пользователя корректно отображает его карму, достижения и активность.
- Данные на странице профиля успешно загружаются из API.
- Написаны тесты, проверяющие корректное отображение данных на странице.

## Ограничения (Constraints)
- Визуальное оформление достижений (иконки, значки) может быть упрощено до текстовых названий в первой итерации.
