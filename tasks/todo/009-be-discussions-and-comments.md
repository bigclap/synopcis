# Обсуждения блоков и древовидные комментарии (Backend)

## Цель (Value)
- **Для системы:** Сделать процесс редактирования более прозрачным и снизить количество "войн право-к", перенося споры из истории изменений в специально отведенное для этого место.

## Требования (Requirements)
1.  **API для комментариев:** Реализовать backend API, которое поддерживает получение древовидной структуры комментариев, пагинацию, отправку, редактирование и удаление комментариев.
2.  **Интеграция с уведомлениями:** Настроить систему так, чтобы пользователи получали уведомления, когда кто-то отвечает на их комментарии.
3.  **Права доступа:** Реализовать ограничения на создание и редактирование комментариев для авторизованных пользователей.

## Критерии приёмки (Acceptance Criteria)
- API для комментариев работает корректно.
- Система уведомлений надежно оповещает пользователей о новых ответах.
- Написаны тесты, покрывающие CRUD-операции с комментариями и логику прав доступа.

## Ограничения (Constraints)
- В рамках этой задачи не реализуется полнофункциональная система модерации комментариев.
