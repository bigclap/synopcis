# Динамическая "Карточка феномена" (Backend)

## Цель (Value)
- **Для системы:** Создать API для управления структурированными данными в карточке феномена, обеспечивая точку входа для структурирования неструктурированных данных.

## Требования (Requirements)
1.  **Статическая генерация:** Обеспечить формирование данных для карточки феномена на стороне сервера во время статической генерации страницы.
2.  **Динамическое обновление:** Реализовать API для асинхронной подгрузки актуальных данных, таких как альтернативные значения свойств и результаты голосования.
3.  **API для редактирования:** Создать API, позволяющий авторизованным пользователям:
    -   Предлагать альтернативные значения для существующих свойств.
    -   Добавлять новые свойства (концепты) в карточку.

## Критерии приёмки (Acceptance Criteria)
- API для обновления и получения данных карточки работает корректно и учитывает права доступа пользователей.
- Реализованы и задокументированы тесты, покрывающие backend (API) часть функционала.

## Ограничения (Constraints)
- В рамках этой задачи не реализуется система модерации предложенных свойств и альтернатив.
