# Просмотр истории блока (Frontend)

## Цель (Value)
- **Для пользователя:** Обеспечить прозрачность и отслеживаемость изменений контента, позволяя видеть, кто, когда и какие правки вносил.

## Требования (Requirements)
1.  **UI для истории:** Создать UI для отображения истории коммитов в модальном окне.
2.  **UI для сравнения:** Реализовать UI для отображения `diff` между двумя выбранными коммитами, наглядно показывая добавленный и удаленный текст.
3.  **Интеграция с API:** Интегрировать UI с backend API для получения истории коммитов и `diff`.

## Критерии приёмки (Acceptance Criteria)
- UI для истории и сравнения корректно отображает данные, полученные от API.
- Пользователь может легко выбрать два коммита для сравнения.
- Написаны тесты, проверяющие frontend-часть функционала.

## Ограничения (Constraints)
- Эта задача охватывает только frontend-часть. Backend-реализация находится в задаче `018-be-view-block-history.md`.
