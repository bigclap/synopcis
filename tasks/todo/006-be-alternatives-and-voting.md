# Конкурирующие альтернативы и голосование (Backend)

## Цель (Value)
- **Для системы:** Создать основу для децентрализованного и объективного отбора наиболее качественного и достоверного контента путем коллективного принятия решений.

## Требования (Requirements)
1.  **Модель данных:** Спроектировать и реализовать в базе данных модель для хранения альтернативных версий блоков и информации о голосах.
2.  **Сохранение структуры:** Обеспечить, чтобы каждая альтернатива блока хранила метаданные (заголовок, уровень вложенности).
3.  **Механизм голосования:** Реализовать API для приема и учета голосов, интегрированный с системой кармы.
4.  **Выбор активной альтернативы:** Реализовать логику для автоматического определения активной альтернативы на основе голосования.

## Критерии приёмки (Acceptance Criteria)
- API для голосования работает корректно, учитывая права и вес голоса.
- Система автоматически и корректно определяет активную альтернативу.
- При смене активной альтернативы сохраняется целостность структуры.
- Написаны тесты, покрывающие логику голосования и выбора активной альтернативы.

## Ограничения (Constraints)
- Сложные сценарии разрешения конфликтов решаются по упрощенной схеме.
