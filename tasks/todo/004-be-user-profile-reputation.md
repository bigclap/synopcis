# Профили пользователей, карма и репутация (Backend)

## Цель (Value)
- **Для системы:** Создать механизм обратной связи, который поощряет качественный вклад и ограничивает деструктивное поведение, тем самым повышая общее качество контента.

## Требования (Requirements)
1.  **Модель кармы:** Спроектировать и реализовать систему подсчета кармы, которая учитывает различные действия пользователя (голоса, правки, жалобы).
2.  **API профиля:** Создать API для получения данных профиля пользователя, включая текущую карму, список достижений и историю активности.
3.  **Система достижений:** Разработать механизм выдачи достижений за определенные действия.
4.  **Права на основе кармы:** Определить и реализовать логику, по которой уровень кармы пользователя влияет на его права и возможности на платформе.

## Критерии приёмки (Acceptance Criteria)
- Карма пользователя корректно пересчитывается при совершении им значимых действий.
- Уровень кармы напрямую влияет на права доступа пользователя.
- API профиля корректно отдает все необходимые данные.
- Механизм выдачи достижений работает автоматически и без сбоев.
- Написаны тесты, проверяющие логику изменения кармы и связанное с ней изменение прав доступа.

## Ограничения (Constraints)
- Сложные механики, такие как "облако интересов" пользователя, выносятся в отдельные задачи.
