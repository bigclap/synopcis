# Просмотр графа знаний (Backend)

## Цель (Value)
- **Для пользователя:** Позволить визуально исследовать связи между различными феноменами и концептами, открывая новые, неочевидные взаимосвязи.
- **Для системы:** Увеличить вовлеченность и время, проводимое пользователями на платформе, за счет предоставления интерактивного инструмента для исследования данных.

## Требования (Requirements)
1.  **API для графа:** Реализовать API-эндпоинт (`GET /api/graph?article=...`), который возвращает данные, необходимые для построения графа связей для указанной статьи.
2.  **Формирование данных:** Данные для графа должны включать:
    -   Центральный узел (текущая статья).
    -   Связанные узлы (другие статьи и концепты).
    -   Информацию о семантической близости для окрашивания узлов.

## Критерии приёмки (Acceptance Criteria)
- API корректно возвращает данные для построения графа.
- Написаны тесты, проверяющие работу API.

## Ограничения (Constraints)
- Эта задача охватывает только backend-часть. Frontend-реализация находится в задаче `021-fe-knowledge-graph.md`.
- В первой итерации граф может быть ограничен одним уровнем вложенности (только прямые связи).
