# Просмотр истории блока (Backend)

## Цель (Value)
- **Для пользователя:** Обеспечить прозрачность и отслеживаемость изменений контента, позволяя видеть, кто, когда и какие правки вносил.
- **Для системы:** Создать механизм аудита и отката изменений, используя возможности Git.

## Требования (Requirements)
1.  **API для истории:** Реализовать API-эндпоинт (`GET /api/git/history?file=...`), который возвращает историю коммитов для указанного файла (блока).
2.  **API для сравнения:** Реализовать API-эндпоинт (`GET /api/git/diff?file=...&commit1=...&commit2=...`), который возвращает `diff` между двумя указанными коммитами.
3.  **Интеграция с Git:** Использовать библиотеку `simple-git` для взаимодействия с Git-репозиторием феномена.

## Критерии приёмки (Acceptance Criteria)
- API для истории корректно возвращает список коммитов.
- API для сравнения корректно возвращает `diff` между коммитами.
- Написаны тесты, проверяющие работу API.

## Ограничения (Constraints)
- Эта задача охватывает только backend-часть. Frontend-реализация находится в задаче `019-fe-view-block-history.md`.
