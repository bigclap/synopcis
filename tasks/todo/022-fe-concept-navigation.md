# Навигация по концептам (Frontend)

## Цель (Value)
- **Для пользователя:** Упростить навигацию и исследование контента, позволяя переходить к страницам, посвященным конкретным концептам.

## Требования (Requirements)
1.  **Кликабельные концепты:** Все концепты, отображаемые на странице (например, в шапке статьи или в карточке феномена), должны быть кликабельными ссылками.
2.  **Страница концепта:** Ссылка должна вести на страницу концепта (например, `/concepts/physics`), которая представляет собой список всех статей, связанных с данным концептом.
3.  **Статическая генерация:** Страницы концептов должны быть статически сгенерированы для обеспечения высокой производительности.

## Критерии приёмки (Acceptance Criteria)
- Концепты являются кликабельными и ведут на корректные страницы.
- Страницы концептов корректно отображают список связанных статей.
- Написаны тесты, проверяющие навигацию по концептам.

## Ограничения (Constraints)
- Эта задача охватывает только frontend-часть. Предполагается, что данные для страниц концептов готовятся на backend-этапе.
