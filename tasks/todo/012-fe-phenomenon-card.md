# Динамическая "Карточка феномена" (Frontend)

## Цель (Value)
- **Для пользователя:** Быстро получать структурированную, ключевую информацию о феномене и влиять на ее содержание через голосование и предложение альтернатив.

## Требования (Requirements)
1.  **Отображение карточки:** Реализовать UI для отображения карточки феномена, которая рендерится на статически сгенерированных страницах.
2.  **Интерфейс редактирования:** Создать UI, позволяющий авторизованным пользователям:
    -   Предлагать альтернативные значения для существующих свойств.
    -   Добавлять новые свойства (концепты) в карточку.
3.  **Интеграция с голосованием:** UI должен быть полностью интегрирован с системой голосования, позволяя пользователям отдавать голоса за предпочитаемые альтернативы.
4.  **Динамическое обновление:** При взаимодействии с карточкой (например, при голосовании) данные должны обновляться динамически без перезагрузки страницы.

## Критерии приёмки (Acceptance Criteria)
- Карточка феномена корректно отображается на статически сгенерированных страницах.
- Авторизованные пользователи могут успешно предлагать альтернативы и добавлять новые свойства через UI.
- Интеграция с API работает корректно.
- Реализованы и задокументированы тесты, покрывающие frontend (UX-сценарии) часть функционала.

## Ограничения (Constraints)
- Стилизация и дизайн карточки должны соответствовать общему дизайн-гайду проекта.
