# Просмотр графа знаний (Frontend)

## Цель (Value)
- **Для пользователя:** Позволить визуально исследовать связи между различными феноменами и концептами, открывая новые, неочевидные взаимосвязи.

## Требования (Requirements)
1.  **UI для графа:** Создать интерактивную визуализацию графа знаний в модальном окне или на отдельной странице.
2.  **Визуализация:**
    -   Текущая статья должна быть в центре графа.
    -   От нее должны идти связи к другим статьям и концептам.
    -   Узлы должны быть окрашены по семантической близости.
3.  **Интеграция с API:** Интегрировать UI с backend API для получения данных графа.
4.  **Библиотеки:** Использовать `D3.js`, `vis.js` или аналогичные библиотеки для визуализации.

## Критерии приёмки (Acceptance Criteria)
- UI корректно отображает граф знаний, полученный от API.
- Визуализация является интерактивной и позволяет пользователю исследовать связи.
- Написаны тесты, проверяющие frontend-часть функционала.

## Ограничения (Constraints)
- Эта задача охватывает только frontend-часть. Backend-реализация находится в задаче `020-be-knowledge-graph.md`.
