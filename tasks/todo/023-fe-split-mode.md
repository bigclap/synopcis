# Сплит-режим (Сравнение перспектив) (Frontend)

## Цель (Value)
- **Для пользователя:** Предоставить уникальную возможность сравнивать различные точки зрения на один и тот же феномен в рамках одного экрана, что способствует более глубокому и объективному пониманию темы.

## Требования (Requirements)
1.  **UI для выбора:** Создать UI, позволяющий пользователю выбрать два концепта для сравнения (например, через модальное окно с выпадающими списками).
2.  **Двухколоночный вид:** После выбора концептов страница должна перестраиваться в двухколоночный вид, где каждая колонка представляет одну из выбранных перспектив.
3.  **Клиентская логика:** Вся логика по сборке и отображению двух версий статьи должна выполняться на стороне клиента. JS-сборщик должен:
    -   Проанализировать `manifest.json`.
    -   Для каждой из двух перспектив найти и загрузить соответствующие `.md` файлы.
    -   Динамически перерисовать контентную область.

## Критерии приёмки (Acceptance Criteria)
- Пользователь может успешно выбрать два концепта для сравнения.
- Страница корректно перестраивается в двухколоночный вид, отображая соответствующие версии контента.
- Вся логика работает на клиенте без необходимости в дополнительных запросах к backend API (кроме загрузки `.md` файлов).
- Написаны тесты, проверяющие работу сплит-режима.

## Ограничения (Constraints)
- Эта задача является чисто frontend-задачей и не требует изменений в backend.
- В первой итерации можно ограничиться поддержкой только двух концептов для сравнения.
